## 任务描述
近年来，前沿语言模型在数学和科学推理任务上取得了显著进展。然而，这类表现通常依赖于非常大的模型规模、大规模预训练以及高推理成本。较小的通用模型在专业知识回忆、定量计算、单位换算、符号操作、多步物理或化学推理等方面仍然更容易失败。

本次作业关注的是：如何系统性地增强一个固定的小型语言模型，使其接近更强模型的问题解决能力。所有排行榜提交所使用的基础模型都是 Qwen3-8B。你的任务是开发方法，使用工具、检索、验证、结构化提示，或其他可复现的系统设计技术，来提升它在大学水平物理和化学填空题上的表现。换句话说，核心研究问题是：外部脚手架如何帮助较小模型缩小一部分能力差距。
### 数据
发布的评测数据集包含 230 道大学水平问题：112 道物理题，118 道化学题。少量条目包含答案作为示例。

每条数据可能包含以下字段：
字段	描述
id	题目编号。
field	学科，可能是 physics 或 chemistry。
question	题目陈述。
answer	答案字段。大多数发布的测试条目中该字段为 null。
subfield	题目的子领域。
theorem	在适用时，表示可能相关的物理定律或原理。
unit	在适用时，表示答案的单位。

评测器使用 math-verify 设计，并支持 LaTeX 风格的数学答案比较；因此，你不需要担心等价数学记号之间的无害差异。

允许使用的外部知识来源。 你只能使用以下教材作为外部知识库：

### 必需 baseline
你必须实现并报告以下 baseline：
模型： Qwen3-8B。
提示方法： 直接 chain-of-thought prompting，即直接思维链提示。
参考准确率： 约 50%。
如果你的 baseline 结果与参考范围差异很大，请在开发更高级方法之前，先调试你的实现、prompt 格式、答案抽取和提交格式。
### 可能的方法
只要你的方法遵守本说明中的数据模型和资源限制，你可以使用任何提示方法或系统设计策略。方法应该用于增强 Qwen3-8B，而不是用更强的模型替代它。示例包括但不限于：
self-consistency，即自洽性方法；
retrieval-augmented generation，即检索增强生成；
agentic pipelines，即智能体式流程；
符号工具、数值工具或代码工具。

你必须在报告中清楚描述完整流程。分析应当严谨且有证据支持：它应该解释你的方法为什么或在哪里失败，量化相对于直接 Qwen3-8B baseline 的提升，识别主要错误类型，比较物理和化学题上的表现，并讨论涉及检索、工具或验证的代表性成功或失败案例。

除此之外，你还必须编写答案抽取脚本。确保将答案抽取为 Python 字符串，因为我们的评测器只接受这种类型。需要注意的是，所有提交的答案字符串都不得包含描述、方程或单位。下面是一些有效和无效答案表达的例子：

有效	无效
3.14	The answer is 3.14^\circ
\frac{22}{\sqrt{7}}	x = \frac{1}{2}mv^2 = 8.32
3e8	3e8 m/s
6.67 \times 10^{-11}	The answer is the gravitational constant
